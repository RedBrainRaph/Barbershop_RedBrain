### 4. POST /api/bookings - создание записи

### 4.1 Успешное создание записи

Команда:
```bash
curl -X POST http://localhost:8080/api/bookings \
  -H "Content-Type: application/json" \
  -d '{
    "client_name": "Тест Тестов",
    "client_phone": "+7 999 888-77-66",
    "client_telegram": "@test",
    "service_id": 1,
    "barber_id": 1,
    "booking_date": "2026-03-15",
    "booking_time": "15:00:00"
  }'
```

Результат: ✅ Успешно (200 OK)

Ответ:
```json
{
  "id": 51,
  "message": "Запись успешно создана"
}
```
Проверки:

Статус код 200

Получен ID новой записи

Сообщение об успехе

---

### 4.2 Попытка записи на занятое время

Команда:
```bash
curl -X POST http://localhost:8080/api/bookings \
  -H "Content-Type: application/json" \
  -d '{
    "client_name": "Тест Тестов",
    "client_phone": "+7 999 888-77-66",
    "client_telegram": "@test",
    "service_id": 1,
    "barber_id": 1,
    "booking_date": "2026-03-15",
    "booking_time": "15:00:00"
  }'
```

Результат: ⚠️ Конфликт (409 Conflict)

Ответ:
```json
{
  "error": "Это время уже занято"
}
```

Проверки:

Статус код 409

Сообщение о конфликте

Запись не создается в БД

---

### 4.3 Невалидные данные (пропущено поле)

```json
  "error": "Неверный формат данных"
}
```

Проверки:

Статус код 400

Сообщение об ошибке валидации
